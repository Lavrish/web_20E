Описание тегов

abbr
Тег определяет аббревиатуру или акроним, например "HTML", 'Mr.", "Dec.", "ASAP", "ATM".
address
Тэг определяет контактную информацию владельца документа или статьи .
aside
Тэг определяет некоторый контент в стороне от содержания.
code
Тег используется для определения куска компьютерного кода. Содержимое внутри отображается моноширинным шрифтом браузера по умолчанию.
caption
Тег определяет заголовок таблицы.Тег должен быть вставлен сразу после тега "table" .
cite
тег определяет название творческой работы (например, книга, стихотворение, песня, кино, живопись, скульптура и т.д.).
fieldset
Тег используется для группировки связанных элементов в форме. Тег рисует рамку вокруг связанных элементов
figcaption
Тэг определяет заголовок для "figure" элемента.
legend
Тэг определяет заголовок для "Fieldset" элемента.
mark
Тэг определяет текст , который должен быть отмечен или выделен.
output
Тег используется для представления результата вычислений (например , один выполняется с помощью скрипта).
optgroup
Тег используется для опций , связанных групп в "select2" элемент (выпадающий список). Если у вас длинный список опций, пользователю будет проще обрабатывать группы связанных опций.
option
Тэг определяет параметр в списке выбора. элементы помещаются внутри элемента "select" , "optgroup" или "datalist" .
picture
Тег дает веб - разработчикам больше гибкости в указании ресурсов изображения. Чаще всего этот "picture" элемент используется для художественного оформления в адаптивном дизайне. Вместо того, чтобы иметь одно изображение, масштабируемое вверх или вниз в зависимости от ширины области просмотра, можно создать несколько изображений, чтобы лучше заполнить область просмотра браузера. "picture" Элемент содержит два тега: один или более "source" тегов и один "img" тега. Браузер будет искать первый элемент "source", в котором медиа-запрос соответствует текущей ширине области просмотра, а затем отобразит правильное изображение (указанное в атрибуте srcset). Элемент "img" требуется как последний дочерний "picture" элемент элемента в качестве запасного варианта, если ни один из исходных тегов не совпадает.
progress
Тег представляет прогресс выполнения задачи.
section
Тег определяет раздел в документе.
samp
Тег используются для определения выборки вывода компьютерной программы. Содержимое внутри отображается моноширинным шрифтом браузера по умолчанию.
summary
Тэг определяет видимый заголовок для details элемента. Заголовок можно щелкнуть, чтобы просмотреть / скрыть подробности.
tfoot
Тег используется для содержания колонтитула группы в качестве HTML - таблицы. В "tfoot"элементе должен быть один или несколько тегов "tr" внутри.
th
Тег определяет ячейку заголовка в HTML - таблице. В таблице HTML есть два типа ячеек: Ячейки заголовка - содержат информацию заголовка (созданного с помощью "th" элемента) Ячейки данных - содержат данные (созданные с помощью элемента "td") Текст в "th" элементах по умолчанию выделен полужирным шрифтом и центрирован. Текст в элементах "td" стандартный и по умолчанию выровнен по левому краю.
thead
Тег используется для содержания заголовка группы в HTML - таблице. Элемент используется в сочетании с "tbody" и "tfoot" элементами , чтобы указать каждую часть таблицы (заголовок, тело, колонтитул). В "thead" элементе должен быть один или несколько тегов "tr" внутри.
time
Тэг определяет время , конкретное (или DateTime). (datetime) Атрибут этого элемента используется перевести время в машиночитаемом формат , так что браузеры могут предложить , чтобы добавить дату напоминания через календарь пользователя, и поисковые системы могут производить умные результаты поиска.
track
тег указывает текст песни для "audio" или "video" элементов. Этот элемент используется для указания субтитров, файлов с субтитрами или других файлов, содержащих текст, которые должны быть видны при воспроизведении мультимедиа. Дорожки отформатированы в формате WebVTT (файлы .vtt).
meter
Тэг определяет скалярное измерение в пределах известного диапазона или дробное значение. Это также известно как датчик. Примеры: использование диска, релевантность результата запроса и т. Д. Тег не должен использоваться для индикации прогресса (как на индикаторе выполнения). Для индикаторов выполнения используйте тег "progress" .
nav
Тег определяет набор навигационных ссылок. Обратите внимание, что НЕ все ссылки документа должны находиться внутри элемента. Элемент предназначен только для основного блока навигационных ссылок.
*menu
Это экспериментальная технология. Эта возможность была удалена из веб-стандартов. Хотя некоторые браузеры по-прежнему могут поддерживать её, она находится в процессе удаления. Не используйте её ни в старых, ни в новых проектах. Страницы или веб-приложения, использующие её, могут в любой момент сломаться.
*menuitem
Эта функция устарела и не должна использоваться.
*keygen
Эта функция устарела и не должна использоваться.
main
Тег определяет основное содержание документа. Содержимое внутри элемента должно быть уникальным для документа. Он не должен содержать какой-либо контент, повторяющийся в документах, например боковые панели, навигационные ссылки, информацию об авторских правах, логотипы сайтов и формы поиска. В документе не должно быть более одного элемента. Элемент не должен быть потомком "article", "aside", "footer", "header" или "nav" элемент.
embed
Тег определяет контейнер для внешнего ресурса, такого как веб - страницу, изображение, медиа - плеер, или плагин в приложении.
